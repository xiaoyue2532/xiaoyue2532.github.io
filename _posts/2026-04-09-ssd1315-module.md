本科在读期间做过用 Arduino 驱动 SSD1306 OLED 屏幕，那时候使用的是现成的 u8g2 库。现在我想用 STM32 驱动 OLED 屏，发现淘宝上在售的 OLED 屏有老款的 SSD1306 和新款的 SSD1315，新款 1315 比 1306 阉割了一部分滚屏功能、价格略低，于是我购入了一块 4针 IIC 接口的 SSD1315。

商家提供的 SSD1315 例程是基于标准库的，本文将在 STM32F103ZET6 和 HK32F103C8T6A 平台上、基于 HAL 库驱动一块 SSD1315 OLED 屏幕。

## 引脚连接

注意 SSD1315 的 VCC 和 GND 排布与 SSD1306 有差别。

| Name | Pin |
| ---- | --- |
| SCL  | PA5 |
| SDA  | PA6 |

## SSD1315 驱动

例程里大量使用 u8, u16, u32 变量类型。分别修改 oled.c 和 oled.h 中的变量类型为 uint8_t, uint16_t, uint32_t。

`oled.c` 中主要要修改初始化相关的函数，标准库和 HAL 库的 GPIO 初始化略有不同：

```c++
// OLED的初始化
void OLED_Init(void)
{
    GPIO_InitTypeDef gpio_init_structure = {0};
    __HAL_RCC_GPIOA_CLK_ENABLE(); // 使能A端口时钟
    gpio_init_structure.Pin = GPIO_PIN_5 | GPIO_PIN_6;
    gpio_init_structure.Mode = GPIO_MODE_OUTPUT_OD;  // 开漏输出
    gpio_init_structure.Speed = GPIO_SPEED_FREQ_HIGH; // 速度50MHz
    HAL_GPIO_Init(GPIOA, &gpio_init_structure);            // 初始化PA5,6
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5 | GPIO_PIN_6, GPIO_PIN_SET);
```

`oled.h` 中 SDL SDA 控制宏也要写成 HAL 库的方式：

```c++
#include "stm32f1xx.h"

#define OLED_SCL_Clr() HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_RESET) // SCL
#define OLED_SCL_Set() HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_SET)

#define OLED_SDA_Clr() HAL_GPIO_WritePin(GPIOA, GPIO_PIN_6, GPIO_PIN_RESET) // SDA
#define OLED_SDA_Set() HAL_GPIO_WritePin(GPIOA, GPIO_PIN_6, GPIO_PIN_SET)
```

## 修改主程序

根据上篇博文，该 `main.c` 适用于 HK32F103C8T6A，对于 STM32F103ZET6 需要将倍频改回 9、延时初始化修改为72。

```c++
#include "./SYSTEM/sys/sys.h"
#include "./SYSTEM/delay/delay.h"
#include "./SYSTEM/usart/usart.h"
#include "./BSP/LED/led.h"
#include "./BSP/OLED/oled.h"
#include "./BSP/OLED/bmp.h"

int main(void)
{
    HAL_Init();                         /* 初始化HAL库 */
    sys_stm32_clock_init(RCC_PLL_MUL15); /* 设置时钟,120M */
    delay_init(120);                     /* 初始化延时函数 */
    led_init();                         /* 初始化LED */

    OLED_Init();
    OLED_ColorTurn(0);   // 0正常显示，1 反色显示
    OLED_DisplayTurn(0); // 0正常显示 1 屏幕翻转显示
    uint8_t t = ' ';

    while (1)
    {
        OLED_ShowPicture(0, 0, 128, 64, BMP1, 1);
        OLED_Refresh();
        delay_ms(500);
        OLED_Clear();
        OLED_ShowChinese(0, 0, 0, 16, 1);   // 中
        OLED_ShowChinese(18, 0, 1, 16, 1);  // 景
        OLED_ShowChinese(36, 0, 2, 16, 1);  // 园
        OLED_ShowChinese(54, 0, 3, 16, 1);  // 电
        OLED_ShowChinese(72, 0, 4, 16, 1);  // 子
        OLED_ShowChinese(90, 0, 5, 16, 1);  // 技
        OLED_ShowChinese(108, 0, 6, 16, 1); // 术
        OLED_ShowString(8, 16, "ZHONGJINGYUAN", 16, 1);
        OLED_ShowString(20, 32, "2014/05/01", 16, 1);
        OLED_ShowString(0, 48, "ASCII:", 16, 1);
        OLED_ShowString(63, 48, "CODE:", 16, 1);
        OLED_ShowChar(48, 48, t, 16, 1); // 显示ASCII字符
        t++;
        if (t > '~')
            t = ' ';
        OLED_ShowNum(103, 48, t, 3, 16, 1);
        OLED_Refresh();
        delay_ms(500);
        OLED_Clear();
        OLED_ShowChinese(0, 0, 0, 16, 1);   // 16*16 中
        OLED_ShowChinese(16, 0, 0, 24, 1);  // 24*24 中
        OLED_ShowChinese(24, 20, 0, 32, 1); // 32*32 中
        OLED_ShowChinese(64, 0, 0, 64, 1);  // 64*64 中
        OLED_Refresh();
        delay_ms(500);
        OLED_Clear();
        OLED_ShowString(0, 0, "ABC", 8, 1);   // 6*8 “ABC”
        OLED_ShowString(0, 8, "ABC", 12, 1);  // 6*12 “ABC”
        OLED_ShowString(0, 20, "ABC", 16, 1); // 8*16 “ABC”
        OLED_ShowString(0, 36, "ABC", 24, 1); // 12*24 “ABC”
        OLED_Refresh();
        delay_ms(500);
        OLED_ScrollDisplay(11, 4, 1);
    }
}
```

### 程序测试

经过实测，相比 STM32F103ZET6 的 72Mhz 主频、HK23F103C8T6A 的 120Mhz 主频更高，HK32 的刷屏就是比 STM32 更快。

该例程不适宜用 AC6 编译。AC6 启用 O1 优化以上，在 STM32F103ZET6 上表现为刷屏变快，但并不是正确的行为、好像是 AC6 把一部分延时优化掉了。而 HK32 上 AC6 编译的程序更是没法运行。
